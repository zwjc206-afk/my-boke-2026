# GitHub 上线链路

## 当前状态

网站已经具备 GitHub Pages 发布链路：

- `main` 分支推送后自动发布网站。
- 每天北京时间 08:00 自动更新 `content/news.generated.json`。
- AI 资讯更新成功提交后，会再次触发网站发布。
- `.nojekyll` 已加入，GitHub Pages 会按静态网站原样发布。

## 你需要先做的一次性动作

在 GitHub 创建一个空仓库。

推荐仓库名：

```text
zwjc206-afk.github.io
```

这样上线地址最短：

```text
https://zwjc206-afk.github.io/
```

如果用其他仓库名，例如：

```text
ai-mentor-site
```

上线地址一般是：

```text
https://zwjc206-afk.github.io/ai-mentor-site/
```

## 推送到 GitHub

拿到空仓库地址后，在当前项目目录运行：

```bash
scripts/publish-to-github.sh "https://github.com/zwjc206-afk/zwjc206-afk.github.io.git"
```

如果你创建的是其他仓库，把地址替换成对应仓库地址即可。

## GitHub Pages 设置

仓库推送成功后，进入 GitHub 仓库：

```text
Settings -> Pages
```

确认：

```text
Source: GitHub Actions
```

通常工作流会自动配置 Pages；如果没有自动开启，手动选择 `GitHub Actions`。

## 验收

1. 打开仓库的 `Actions` 页面。
2. 确认 `Deploy static site to GitHub Pages` 运行成功。
3. 打开 GitHub Pages 给出的线上网址。
4. 检查首页图片、讲课视频、合作企业、中文 AI 资讯、联系方式是否正常。

## 后续维护

本地改完内容后：

```bash
git add .
git commit -m "Update site content"
git push
```

GitHub 会自动发布新版。

每天 AI 资讯无需手动更新；GitHub Actions 会定时执行：

```bash
node scripts/update-ai-news.mjs
```
